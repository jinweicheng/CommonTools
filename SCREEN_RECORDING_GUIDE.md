# 🎥 iPhone 屏幕录像处理 - 快速使用指南

## ✨ 核心功能

### ✂️ 智能裁剪
去除顶部红点、时间戳、状态栏和底部 Home bar，让录屏画面更干净专业。

### 🗜️ 高效压缩
平均压缩 70%+ 体积，方便邮件发送和社交媒体分享，同时保持优秀的视觉质量。

### 👁️ 隐私保护
模糊或遮挡敏感区域（通知、消息预览、个人信息），安全分享。

### ⚡ 本地处理
使用 FFmpeg WebAssembly 技术，所有处理在浏览器中完成，视频不上传服务器。

---

## 🎯 快速开始

### 1. 上传视频 📤

**方式 1：点击上传**
```
点击"上传屏幕录像"按钮
→ 选择 .mov 或 .mp4 文件
→ 支持多文件选择
```

**方式 2：拖拽上传**
```
拖动视频到上传区域
→ 看到紫色高亮和弹跳动画
→ 松开鼠标完成上传
```

**文件要求：**
- ✅ 格式：.MOV, .MP4
- ✅ 大小：< 500MB
- ✅ 时长：< 10 分钟
- ✅ 设备：桌面浏览器（Chrome/Edge）

---

### 2. 选择处理类型 🎬

```
┌──────────┬──────────┬──────────┬──────────┐
│✂️ 仅裁剪 │🗜️ 仅压缩│👁️ 仅模糊│📦 全部  │
└──────────┴──────────┴──────────┴──────────┘
```

**推荐选择：**

| 场景 | 推荐 | 说明 |
|------|------|------|
| **教程制作** | 📦 全部 | 裁剪+压缩，最专业 |
| **快速分享** | 🗜️ 仅压缩 | 减小体积，快速发送 |
| **隐私保护** | 👁️ 仅模糊 | 遮挡敏感信息 |
| **清理画面** | ✂️ 仅裁剪 | 去除状态栏 |

---

### 3. 配置裁剪参数 ✂️

**iPhone 标准预设（推荐）：**
```
顶部: 120px  ← 去除红点、时间、状态栏
底部: 80px   ← 去除 Home bar
左侧: 0px
右侧: 0px

[iPhone 预设] ← 一键设置
```

**自定义裁剪：**
- 手动输入像素值
- 适配不同 iPhone 型号
- 实时预览效果

**不同机型参考：**
```
iPhone 14 Pro Max:
顶部: 140px（动态岛更高）
底部: 80px

iPhone SE:
顶部: 100px（无动态岛）
底部: 60px
```

### 技巧 2：压缩质量选择

```
用途推荐：
邮件发送：中/低  （< 25MB）
微信分享：中/低  （< 25MB）
专业演示：高      （保持画质）
内部沟通：低      （速度优先）
```

---

### 技巧 3：分批处理大量视频

```
建议：
- 每批 3-5 个视频

- 清除后再上传下一批
- 避免浏览器内存不足
```

---

### 技巧 4：性能优化建议

```
最佳实践：
✅ 关闭其他标签页
✅ 使用桌面浏览器
✅ 视频时长 < 5 分钟
✅ 文件大小 < 200MB
✅ 处理完成后清除

避免：
❌ 移动端浏览器
❌ 同时处理多个长视频
❌ 超过 10 分钟的视频
❌ 超过 500MB 的文件
```

## ⚠️ 注意事项

### 1. 首次使用需要等待

```
首次加载 FFmpeg（1-2分钟）
├─ 下载 ffmpeg-core.js（~5MB）
├─ 下载 ffmpeg-core.wasm（~25MB）
└─ 初始化 WebAssembly

后续使用（浏览器缓存）
└─ 几秒内加载完成 ⚡
```

---

### 2. 处理时间参考

```
1 分钟视频：30-60 秒
3 分钟视频：1.5-3 分钟
5 分钟视频：2.5-5 分钟
10 分钟视频：5-10 分钟
```

**影响因素：**
- 视频时长
- 处理类型（全部 > 单独）
- 电脑性能
- 压缩质量

## 📞 技术支持

### 遇到问题？

1. **FFmpeg 加载失败**
   - 检查网络连接
   - 刷新页面重试
   - 清除浏览器缓存

2. **处理卡死**
   - 视频是否过大？
   - 其他标签页是否关闭？
   - 尝试降低压缩质量

3. **输出无法播放**
   - 使用最新播放器
   - 转换为 MP4 格式
   - 检查原视频是否损坏
